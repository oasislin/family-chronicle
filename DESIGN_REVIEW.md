# 家族编年史 Web 界面优化设计方案

## 一、现状分析

### 现有能力
| 模块 | 功能 | 状态 |
|------|------|------|
| 图谱可视化 | 节点展示、点击选中、网格布局 | ✅ 基本可用 |
| 人物详情 | 查看信息、时间线、关系列表 | ✅ 只读，无编辑 |
| AI 文本输入 | 解析人物/事件/关系 | ✅ 可用 |
| 待办校正台 | 入库/冲突处理 | ⚠️ 部分功能 |
| 关系管理 | 无入口 | ❌ 缺失 |
| 事件管理 | 无入口 | ❌ 缺失 |

### 核心痛点
1. **只能看不能改** — 所有信息展示为只读，无法编辑人物/关系/事件
2. **关系只能靠 AI** — 没有手动创建关系的入口
3. **冲突处理不直观** — "合并到现有人员"没有选择器，"语义模糊"项无可操作界面
4. **图谱太简单** — 没有关系类型筛选、层级展开、聚焦高亮
5. **AI 解析结果无法批量操作** — 人物、事件、关系需要分别入库

### 改动4：后端 API 补全

当前后端缺少以下端点，需要补充：

```
PUT  /api/families/{familyId}/people/{personId}       # 更新人物
POST /api/families/{familyId}/relationships/{relId}   # 创建/更新关系
DELETE /api/families/{familyId}/relationships/{relId} # 删除关系
PUT  /api/families/{familyId}/events/{eventId}        # 更新事件
POST /api/families/{familyId}/events/{eventId}/participants # 添加参与者
```

前端 `api.ts` 对应新增：
```typescript
personApi.update(familyId, personId, data)
personApi.delete(familyId, personId)
relationshipApi.create(familyId, data)
relationshipApi.delete(familyId, relId)
eventApi.update(familyId, eventId, data)
```

## 五、补充设计（v2）

### 5.1 人物故事（Story）

**场景**：「王建国年轻的时候很帅，读书很好」「赵大爷是村里出了名的热心肠」

这类信息不包含关系、不是事件，是自由叙事。需要一个独立的 story 字段。

**数据模型变更**：
```
Person.story: string  // 自由文本，支持多段落
```

**前端展示**（PersonDetail Tab 中新增「生平故事」Tab）：
```
【生平故事 Tab】
┌─────────────────────────────────────────┐
│  📖 王建国的生平故事                      │
│                                         │
│  王建国年轻的时候长得一表人才，读书成绩     │
│  在班上一直是前几名。高中毕业后没考上大学， │
│  回村学了木匠手艺，后来在县城开了家具店。   │
│                                         │
│  [✏️ 编辑故事]                            │
│                                         │
│  编辑模式:                               │
│  ┌─────────────────────────────────┐     │
│  │ [多行文本编辑器，支持 Markdown]    │     │
│  │                                 │     │
│  │                                 │     │
│  └─────────────────────────────────┘     │
│  [💾 保存] [❌ 取消]                      │
└─────────────────────────────────────────┘
```

**后端变更**：
- `Person` model 增加 `story: Optional[str]`
- `PersonCreate` 增加 `story` 字段
- 新增 `PUT /api/families/{familyId}/people/{personId}` 端点（支持更新 story）

**AI 解析联动**：
- AI 解析结果中如果有非关系、非事件的描述性内容，自动填入 `story` 字段
- 例如输入「王建国年轻时很帅，读书很好」→ AI 不提取人物/事件，而是将此文本归类为 story

### 5.2 存储架构（纯 JSON）

**决策**：移除 Neo4j 依赖，JSON 文件作为唯一存储。

**理由**：
- 家族数据规模 50-500 人，JSON 完全够用
- 避免维护同步层、额外进程、故障降级等复杂度
- 图查询（祖先/后代/最短路径）用代码递归/BFS 实现即可

**数据流**：
```
前端请求 → main.py → JSON 文件（唯一存储）
```

**图查询用代码实现**：
```python

    ...
```

### 5.3 清理 Neo4j / Docker / MongoDB 相关内容

**需删除的文件/目录**：
- `backend/neo4j_service.py` — Neo4j 服务模块
- `docker-compose.yml` — Docker 编排文件
- `neo4j-init/` — Neo4j 初始化脚本目录
- `mongo-init/` — MongoDB 初始化脚本目录

**需清理的配置**：
- `config.py` — 移除 NEO4J_URI/USER/PASSWORD、MONGODB_URI/DATABASE
- `config.py` — 移除 `get_database_config()` 函数
- `requirements.txt` — 移除 `neo4j==5.14.0`、`pymongo==4.6.0`

**需停止的服务**：
- `brew services stop neo4j`
- `brew uninstall neo4j`（可选，用户确认）

**需卸载的 Python 包**（venv 内）：
- `neo4j`、`pymongo`、`motor`
