# 家族编年史 — 操作逻辑 v3（定稿）

## 设计哲学

> **用户只做一件半事。**
> - **一件事**：输入文字（或语音），告诉系统家族故事。
> - **半件事**：系统不确定时，回答一个具体的提问。
>
> 系统是一个**倾听者和整理者**。用户说完，家谱就更新了。

## 消息流（右栏）设计

消息流像一个**系统日志**，记录系统做了什么。只有以下情况才有内容：

| 消息类型 | 触发条件 | 展示内容 |
|---------|---------|---------|
| ✅ 成功入库 | 解析完成且无歧义 | 简要清单（新增/更新了什么） |
| ⚠️ 需要确认 | 有歧义需要人工判断 | 具体问题 + 选项按钮 |
| ❌ 解析失败 | AI 出错 | 错误信息 + 重试按钮 |
| 💬 追问建议 | AI 检测到信息不完整 | 自然语言追问（可忽略） |

**消息流特性**：
- 每条消息是独立的卡片
- 历史消息可滚动查看
- 成功消息自动折叠（3秒后收起为一行摘要）
- 需要确认的消息**不会自动消失**，直到用户处理
- 底部始终有输入框

## 需要用户确认的 4 种场景

### 场景1：人物匹配歧义
AI 识别到「建国」，数据库中有「王建国」「李建国」
```
⚠️ 你提到的「建国」，是以下哪位？
  [👨 王建国] [👨 李建国] [➕ 是新人物]
```

### 场景2：逻辑矛盾
用户说「王建国1980年出生」，但数据库中记录为1975年
```
⚠️ 王建国的出生日期有冲突：
  数据库记录: 1975年
  你刚才提到: 1980年
  
  [以新信息为准] [保留原记录]
```

### 场景3：AI 置信度低
AI 不确定某个信息的可靠性
```
⚠️ 以下信息不太确定，请确认：
  「赵大爷是村长」—— 是否将赵大爷标记为村长？
  
  [确认] [跳过]
```

### 场景4：关系方向不明
AI 知道 A 和 B 有亲子关系，但不确定谁是父谁是子
```
⚠️ 王大强和王建国的亲子关系方向：
  [王大强是父亲 → 王建国] 
  [王建国是父亲 → 王大强]
```

## AI 解析后的自动处理流程

```
AI 返回 { entities, events, relationships, metadata }
  │
  ├─ metadata.ambiguous_references 为空？
  │   ├─ YES → 直接全部入库 → 显示 ✅ 消息
  │   └─ NO  → 进入确认流程
  │
  ├─ 人物自动匹配（姓名精确/模糊匹配）
  │   ├─ 唯一匹配 → 自动关联
  │   ├─ 多个匹配 → ⚠️ 提问
  │   └─ 无匹配   → 自动创建
  │
  ├─ 关系自动去重
  │   ├─ 已存在 → 跳过（不提示）
  │   └─ 不存在 → 自动创建
  │
  ├─ 事件自动创建
  │   └─ 自动关联参与者（用匹配后的真实ID）
  │
  └─ metadata.suggested_questions？
      └─ 以自然语言形式追加到消息流（可忽略）
         💬 "对了，建国和李梅结婚具体是哪一年？"
```

### 关键后端逻辑：`auto-import` 端点

```python
@app.post("/api/families/{family_id}/auto-import")
async def auto_import(family_id: str, parsed_data: dict):
    """自动导入：匹配、去重、入库，返回操作日志和需要确认的问题"""
    graph = load_family_graph(family_id)
    actions = []      # 操作日志
    questions = []    # 需要用户确认的问题
    need_confirm = False

    # 1. 人物处理
    id_mapping = {}   # temp_id → real_id
    for entity in parsed_data.get("entities", []):
        matches = fuzzy_match(entity["name"], graph.people.values())
        
        if len(matches) == 1 and matches[0].score > 0.85:
            # 唯一匹配 → 自动关联
            id_mapping[entity["temp_id"]] = matches[0].person.id
            # 合并新标签
            if entity.get("tags"):
                merge_tags(matches[0].person, entity["tags"])
            actions.append(f"关联人物: {entity['name']} → {matches[0].person.name}")
            
        elif len(matches) > 1:
            # 多个匹配 → 需要确认
            need_confirm = True
            questions.append({
                "type": "person_match",
                "entity": entity,
                "candidates": [m.person.to_dict() for m in matches],
                "temp_id": entity["temp_id"],
            })
            
        else:
            # 无匹配 → 自动创建
            person = create_person_from_entity(entity)
            graph.add_person(person)
            id_mapping[entity["temp_id"]] = person.id
            actions.append(f"新增人物: {person.name}")

    # 2. 关系处理（用真实 ID）
    for rel in parsed_data.get("relationships", []):
        p1_id = id_mapping.get(rel["person1_temp_id"])
        p2_id = id_mapping.get(rel["person2_temp_id"])
        
        if not p1_id or not p2_id:
            continue
            
        # 去重检查
        is_dup = any(
            (r.person1_id == p1_id and r.person2_id == p2_id and r.type == rel["type"])
            or (r.person1_id == p2_id and r.person2_id == p1_id and r.type == rel["type"])
            for r in graph.relationships.values()
        )
        if is_dup:
            continue  # 静默跳过
            
        relationship = create_relationship(p1_id, p2_id, rel)
        graph.add_relationship(relationship)
        actions.append(f"新增关系: {rel['type']}")

    # 3. 事件处理
    for event in parsed_data.get("events", []):
        ev = create_event_from_parsed(event, id_mapping)
        graph.add_event(ev)
        actions.append(f"新增事件: {ev.description}")

    # 4. 保存
    if not need_confirm:
        save_family_graph(family_id, graph)
    
    return {
        "auto_saved": not need_confirm,
        "actions": actions,
        "questions": questions,
    }
```
